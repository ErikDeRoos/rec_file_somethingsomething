# 1_simple_singlefile

## Purpose
This is the first shared example used by both documentation and tests.

## What it demonstrates
- A single `.rec` file
- A typed record set using `%rec: User`
- A comment header line
- Two simple records
- Multiline `Notes` field values using `+`

## Files
- `user.rec` - the first tangible example rec file in the repository

## Usage
- Refer to this example from DLL documentation
- Use this example as direct test input from the test project
- Use this example as the first validation target for parser and serializer work
